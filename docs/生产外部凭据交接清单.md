# 生产外部凭据交接清单

本清单只记录真实联调需要的外部输入，不要求把密钥写进仓库。

## 管理后台

- `ADMIN_INITIAL_USERNAME`：首次管理员账号。
- `ADMIN_INITIAL_PASSWORD`：首次管理员密码，放 Edgespark secret。
- `ADMIN_PASSWORD_PEPPER`：后台密码加盐，放 Edgespark secret。

生产验收提供 `XIABI_VERIFY_ADMIN_USERNAME` / `XIABI_VERIFY_ADMIN_PASSWORD` 后，会登录后台、读取系统自检和带分页的运营列表，并等值保存当前后台配置后读取公开配置，确认后台配置能影响用户端配置读取。

## DeepSeek 写信

- `LETTER_PROVIDER=deepseek`
- `DEEPSEEK_MODEL=deepseek-v4-pro`
- `DEEPSEEK_BASE_URL=https://api.deepseek.com`
- `DEEPSEEK_API_KEY`：放 Edgespark secret。

生产验收：

```powershell
$env:XIABI_VERIFY_DEEPSEEK="1"
npm run verify:production
```

## 微信支付

- `PAYMENT_PROVIDER=wechat`
- `PUBLIC_BASE_URL=https://immortal-sponge-1728.edgespark.app`
- `PAYMENT_NOTIFY_URL=https://immortal-sponge-1728.edgespark.app/api/webhooks/wechat-pay`
- `WECHAT_PAY_APP_ID`：微信支付下单使用的 AppID。
- `WECHAT_MP_APP_ID`：公众号网页授权/微信内 JSAPI 支付使用的 AppID；如果和 `WECHAT_PAY_APP_ID` 相同可以不填，系统会自动复用。
- `WECHAT_PAY_MCH_ID`：微信支付商户号。
- `WECHAT_PAY_PRIVATE_KEY`：商户 API 私钥，放 Edgespark secret。
- `WECHAT_PAY_CERT_SERIAL_NO`：商户 API 证书序列号，放 Edgespark secret。
- `WECHAT_PAY_API_V3_KEY`：API v3 密钥，放 Edgespark secret。
- `WECHAT_PAY_PLATFORM_PUBLIC_KEY`：可选。微信支付平台公钥，放 Edgespark secret；未配置时，服务端会用商户凭据调用微信 `/v3/certificates` 自动拉取平台证书完成回调验签。
- `WECHAT_PAY_PLATFORM_CERT_SERIAL_NO`：可选。指定平台证书序列号，放 Edgespark secret；不填时按微信回调头里的序列号自动匹配。
- `WECHAT_MP_APP_SECRET`：公众号网页授权 secret，微信内 JSAPI 支付需要，放 Edgespark secret；缺少时微信内下单会返回 `wechat_oauth_not_configured`，不会继续生成无法完成回调的授权链接。

生产验收分两步：

```powershell
$env:XIABI_VERIFY_PAYMENT_CREATE="1"
npm run verify:production
```

然后在微信内完成一笔真实小额付款，确认订单变为已支付、权益流水生成、重复回调不重复发权益。

真实付款完成后，用订单 ID 复验订单和权益闭环：

```powershell
$env:XIABI_VERIFY_ADMIN_USERNAME="后台账号"
$env:XIABI_VERIFY_ADMIN_PASSWORD="后台密码"
$env:XIABI_VERIFY_PAID_ORDER_ID="已完成付款的订单ID"
$env:XIABI_VERIFY_REQUIRE_WEBHOOK="1"
npm run verify:production
```

这一步还会连续调用两次后台“补权益”接口，确认已支付订单的权益补发是幂等的，不会重复增加权益流水。

如果某次验收只想证明查单补偿后的订单和权益已到账，可以暂时不设置 `XIABI_VERIFY_REQUIRE_WEBHOOK`；正式支付验收建议保持为 `1`。

## 阿里云短信

- `SMS_PROVIDER=aliyun`
- `SMS_ALIYUN_SIGN_NAME`：短信签名。
- `SMS_ALIYUN_TEMPLATE_CODE`：短信模板 Code。
- `SMS_API_KEY`：阿里云 AccessKey，放 Edgespark secret。
- `SMS_API_SECRET`：阿里云 AccessSecret，放 Edgespark secret。

生产验收：

```powershell
$env:XIABI_VERIFY_SMS_PHONE="可接收验证码的手机号"
npm run verify:production

$env:XIABI_VERIFY_SMS_CODE="收到的6位验证码"
npm run verify:production
```

如果同一轮同时设置 `XIABI_VERIFY_DEEPSEEK=1`，脚本会先生成并领取一封信，再用验证码绑定手机号，确认这封信和首次免费权益都归属到绑定后的用户。

## MiniMax 语音

- `VOICE_PROVIDER=minimax`
- `VOICE_API_KEY`：MiniMax API Key，放 Edgespark secret。
- `MINIMAX_GROUP_ID`：MiniMax Group ID，例如 `2000472756147200305`，放 Edgespark var。
- `MINIMAX_VOICE_ID`：智多星说话音色，例如 `moss_audio_7a15101b-4f96-11f1-b3fb-821d30892425`。
- `MINIMAX_TTS_ENDPOINT`：可选。默认使用 `https://api.minimax.io/v1/t2a_v2`，代码会自动尝试国际站加速端点和国内站端点。
- `MINIMAX_TTS_OUTPUT_FORMAT`：可选，默认 `hex`；如果账号端点支持音频 URL，也可以设为 `url`。
- `MINIMAX_TTS_MODEL`：可选，默认 `speech-2.8-hd`。

如果验收返回 `invalid api key`，优先检查 Key 是否来自当前选择的 MiniMax 站点/计费类型，并按需设置 `MINIMAX_TTS_ENDPOINT`。

生产验收：

```powershell
$env:XIABI_VERIFY_TTS="1"
npm run verify:production
```

## 语音输入 ASR

当前用户端优先使用浏览器语音识别；浏览器不支持时，服务端会把录音转发到可配置 ASR 接入位。

- `VOICE_ASR_PROVIDER`：供应商标识。
- `VOICE_ASR_ENDPOINT`：服务端 ASR HTTP 接口。
- `VOICE_ASR_MODEL`：可选模型名。
- `VOICE_ASR_REQUEST_FORMAT`：可选，`json` 或 `openai`；不填时如果 endpoint 包含 `/audio/transcriptions` 会自动按 OpenAI-compatible multipart 发送，否则按 JSON base64 发送。
- `VOICE_INPUT_MODE`：可选；填 `server` 时，用户端按住说话优先录音并提交服务端 ASR，不再优先使用浏览器内置语音识别。
- `VOICE_ASR_VERIFIED=1`：真实音频样本验收通过后再设置；未设置时用户端不会把服务端录音转写当成已可用能力。
- `VOICE_ASR_API_KEY`：ASR Key，放 Edgespark secret；如果 ASR 和 TTS 共用 Key，也可以复用 `VOICE_API_KEY`。
如果 `VOICE_ASR_PROVIDER=minimax` 或 endpoint 域名包含 `minimax`，服务端会自动把 `MINIMAX_GROUP_ID` 追加到请求 URL；JSON 格式请求也会同时携带 `group_id` / `GroupId` 字段。2026-05-28 重新读取 MiniMax 官方文档索引 `https://platform.minimax.io/docs/llms.txt`，Speech 相关 API 仍未列出独立 ASR/STT/Transcription 接口；只有拿到具体接口并通过样本验收后，才应设置 `VOICE_ASR_VERIFIED=1`。

ASR 返回值兼容：服务端会从纯文本响应、`transcript`、`text`、`asr_text`、`result_text`、`output_text`、`data.transcript`、`data.text`、`data.result.text`、`result.text`、`output.text`、`segments[].text`、`utterances[].text`、OpenAI-compatible `choices[0].text` / `choices[0].message.content` 中读取识别文本。若仍未读到文本，会返回业务错误，不会伪造识别内容。

生产验收：

```powershell
$env:XIABI_VERIFY_ASR_AUDIO="D:\path\to\sample.wav"
$env:XIABI_VERIFY_ASR_EXPECTED_TEXT="样本音频里说出的关键句"
npm run verify:production
```

## 验收结论口径

- 没有真实 verifier 环境变量时，只能证明基础线上运行正常。
- 完整真实运行状态必须完成：后台登录、DeepSeek 生成、短信绑定、微信付款回调发权益、MiniMax TTS、ASR 音频样本。
