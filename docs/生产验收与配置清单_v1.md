# 生产验收与配置清单 v1

更新时间：2026-05-27

## 线上地址

- 用户端 H5：`https://immortal-sponge-1728.edgespark.app/index.html`
- 管理后台：`https://immortal-sponge-1728.edgespark.app/admin.html`
- 健康检查：`https://immortal-sponge-1728.edgespark.app/api/public/health`

## 已具备的真实能力

- 用户端首页、通话引导、打字模式、生成页、信件页、记录页、订单页、设置页均可交互。
- 后台首页文案、通话引导、销售信模板、价格权益、支付开关会真实影响用户端。
- 生产验收提供后台账号后，会核对首页文案、单封/年卡价格、支付开关、系统开关和通话阶段配置是否真实出现在用户端公开配置里。
- 写信生成走服务端 DeepSeek；DeepSeek 不可用时任务会失败，不再生成本地兜底信件。
- 写信任务创建后先进入 `queued`，用户端通过轮询推进任务；任务成功或失败都会落库，刷新页面后仍可查询。
- 首次免费、单封解锁、年卡权益都写入权益流水；前端不能直接决定权限。
- 单封解锁必须关联当前会话的一封销售信。
- 导出打印版前会校验领取、单封或年卡权益。
- 微信 H5 支付下单、支付回调验签/解密、订单置 paid、权益发放、后台查单和补发权益已接入；支付成功必须完整匹配订单号、交易号、AppID、商户号、金额和币种后才会发放权益。
- 微信内打开时已预留 JSAPI 支付链路：没有 openid 会先返回公众号授权地址，授权后使用 JSAPI 参数拉起微信支付；普通手机浏览器仍走 H5 支付。
- 阿里云短信发送、验证码校验、手机号绑定已接入；短信未配置时会明确失败。
- 手机号绑定会同步迁移同会话下的信件、任务、订单、权益、产品档案和导出文件归属，避免用户绑定后资产分散在游客会话里。
- 微信支付配置不完整时不会创建待支付订单；短信验证码错码达到上限会锁定，需要重新获取。
- 支付拉起失败的订单会标记为 `payment_failed`，用户端可在订单页重新支付，后台概览会提示支付未完成订单。
- MiniMax TTS 已接入说话能力；用户端语音输入优先使用浏览器语音识别，微信内 H5 可在公众号 JS-SDK 配置齐全后使用微信录音识别，不支持时可录音提交到服务端 ASR 接入槽。
- 后台有用户、产品档案、信件、任务、订单、权益、支付回调、反馈、审计日志列表和详情；运营列表支持分页，失败任务可重试，且重试会遵守写信开关并防并发重复触发。
- 用户端记录、订单、我的和档案页在资产同步失败时会展示提示；生成失败会保留服务端返回的业务原因，便于用户重试或反馈。
- 后台有系统自检页，可检查 DeepSeek、微信支付、短信、MiniMax 说话、语音转写、管理员安全和公网地址是否配置齐全；自检只显示状态，不显示密钥内容。

## 必填配置

### DeepSeek

- `LETTER_PROVIDER=deepseek`
- `DEEPSEEK_MODEL=deepseek-v4-pro`
- `DEEPSEEK_BASE_URL=https://api.deepseek.com`
- Secret：`DEEPSEEK_API_KEY`

### MiniMax 说话

- `VOICE_PROVIDER=minimax`
- `MINIMAX_GROUP_ID=2000472756147200305`
- `MINIMAX_VOICE_ID=moss_audio_7a15101b-4f96-11f1-b3fb-821d30892425`
- 可选：`MINIMAX_TTS_ENDPOINT`
- 可选：`MINIMAX_TTS_OUTPUT_FORMAT=hex`
- 可选：`MINIMAX_TTS_MODEL=speech-2.8-hd`
- Secret：`VOICE_API_KEY`

说明：MiniMax TTS 会默认请求 HTTP T2A 接口并使用 `hex` 音频输出，代码会自动尝试国际站、国际站加速端点和国内站端点。2026-05-28 重新读取 MiniMax 官方文档索引 `https://platform.minimax.io/docs/llms.txt`，Speech 相关 API 仍是 T2A HTTP、T2A WebSocket、T2A Async、Voice Cloning、Voice Design、Voice Management，未列出独立 STT/ASR/Transcription 转写接口。语音输入如果必须走 MiniMax，需要 MiniMax ASR/转写接口文档或后台开通说明；项目当前不会臆造 MiniMax 转写 URL。

### 语音输入转写接入槽

- `VOICE_ASR_PROVIDER`
- `VOICE_ASR_ENDPOINT`
- `VOICE_ASR_MODEL`
- `VOICE_ASR_REQUEST_FORMAT`：可选，`json` 或 `openai`；不填时如果 endpoint 包含 `/audio/transcriptions` 会自动按 OpenAI-compatible multipart 发送。
- `VOICE_INPUT_MODE`：可选，填 `server` 时按住说话优先走服务端 ASR；不填时浏览器支持直接语音识别则优先用浏览器能力。
- `VOICE_ASR_VERIFIED=1`：真实音频样本验收通过后再设置；未设置时用户端不会把服务端录音转写当成已可用能力。
- Secret：`VOICE_ASR_API_KEY`

说明：浏览器不支持直接语音识别，或 `VOICE_INPUT_MODE=server` / `VOICE_ASR_PROVIDER=minimax` 时，用户端会在 `VOICE_ASR_VERIFIED=1` 后录音提交到 `/api/public/voice/transcribe`。服务端会把录音提交给 `VOICE_ASR_ENDPOINT`；默认支持 JSON base64，也支持 OpenAI-compatible `/audio/transcriptions` multipart 格式，并从纯文本响应、`transcript`、`text`、`asr_text`、`data.result.text`、`segments[].text`、`utterances[].text`、OpenAI-compatible `choices[0].text` / `choices[0].message.content` 等常见返回结构读取识别文本。未配置或未验收时会明确提示切换打字模式，不会伪造识别结果。

### 微信内 H5 语音输入

- `WECHAT_MP_APP_ID`：公众号 AppID；不填时复用 `WECHAT_PAY_APP_ID`。
- `WECHAT_MP_APP_SECRET`：公众号 secret，放 Edgespark secret。
- `PUBLIC_BASE_URL`：必须是当前 Edgespark/正式域名。

说明：微信内打开 H5 时，用户端会在公开配置显示 `capabilities.voice.wechatJssdkConfigured=true` 后优先调用微信 JS-SDK 的 `startRecord`、`stopRecord` 和 `translateVoice` 完成按住说话识别。该链路还需要在微信公众平台把当前域名配置为 JS 接口安全域名；缺配置时会降级到打字或其他可用语音通道。

### 微信支付 H5

- `PAYMENT_PROVIDER=wechat`
- `WECHAT_PAY_APP_ID`
- 可选：`WECHAT_MP_APP_ID`（公众号网页授权/微信内 JSAPI 支付 AppID；不填时复用 `WECHAT_PAY_APP_ID`）
- `WECHAT_PAY_MCH_ID`
- `PAYMENT_NOTIFY_URL=https://immortal-sponge-1728.edgespark.app/api/webhooks/wechat-pay`
- `PUBLIC_BASE_URL=https://immortal-sponge-1728.edgespark.app`
- Secret：`WECHAT_PAY_PRIVATE_KEY`
- Secret：`WECHAT_PAY_CERT_SERIAL_NO`
- Secret：`WECHAT_PAY_API_V3_KEY`
- 可选 Secret：`WECHAT_PAY_PLATFORM_PUBLIC_KEY`
- 可选 Secret：`WECHAT_PAY_PLATFORM_CERT_SERIAL_NO`
- Secret：`WECHAT_MP_APP_SECRET`（微信内 JSAPI/openid 授权需要）

说明：普通手机浏览器走 H5 支付；微信内打开走公众号 `snsapi_base` 授权拿 openid 后再走 JSAPI 支付。公众号授权现在要求 `WECHAT_MP_APP_SECRET` 和 `PUBLIC_BASE_URL` 齐全，并会校验带会话绑定、10 分钟过期和 HMAC 签名的 `state`，缺配置时会直接返回 `wechat_oauth_not_configured`。如果公众号 AppID 与支付下单 AppID 不同，填写 `WECHAT_MP_APP_ID`；未填写时默认复用 `WECHAT_PAY_APP_ID`。回调验签优先使用手动配置的平台公钥，未配置时会用商户凭据自动拉取微信 `/v3/certificates` 平台证书。支付目录、网页授权域名和回调域名需要在微信公众平台/商户平台配置到当前 Edgespark 域名。

当前线上支付创建已能真实请求微信支付；若返回“商户号该产品权限未开通”，需要在微信商户平台产品中心开通 H5 支付，或在微信内场景改用 JSAPI 支付并补齐公众号网页授权配置。

## 生产验收步骤

1. 后台登录后，保存首页文案、引导阶段、销售信模板和价格配置。
2. 打开后台“系统自检”，确认必填项没有“缺配置”；如果有缺项，先补 Edgespark 变量或密钥。
3. 用户端打开首页，确认后台文案已同步。
4. 完成一次打字或语音识别通话，生成销售信。
5. 绑定真实手机号，确认短信能收到且验证码可绑定。
6. 领取首次免费信件，确认权益流水出现 `first_free_letter`。
7. 再生成一封信，确认不能重复首次免费领取。
8. 创建单封解锁订单，完成微信支付，确认回调后订单为 `paid`，权益流水出现 `single`。
9. 创建年卡订单，完成微信支付，确认权益流水出现 `annual`。
10. 订单回调延迟时，在用户端订单页点“刷新支付结果”，在后台订单详情点“查单”。
11. 对已支付但权益缺失的订单，在后台点“补权益”，确认幂等不重复发放。
12. 打开信件导出，确认未领取/未解锁不能导出，已领取/已解锁/年卡可打开打印版。
13. 在后台查看用户、信件、任务、订单、权益、支付回调和审计日志详情。

## 命令行强验收

基础冒烟仍使用：

```bash
npm run verify:live
```

真实外部链路使用：

```bash
npm run verify:production
```

默认只检查健康接口、公开配置，并在提供管理员账号时检查后台系统自检。需要真实调用外部服务时显式打开：

- `XIABI_VERIFY_ADMIN_USERNAME` / `XIABI_VERIFY_ADMIN_PASSWORD`：登录后台、读取系统自检和带分页的运营列表，并等值保存当前后台配置后读取公开配置，确认后台配置能影响用户端配置读取。
- `XIABI_PRODUCTION_STRICT=1`：后台自检有必填缺项时直接失败。
- `XIABI_VERIFY_DEEPSEEK=1`：真实调用 DeepSeek 生成一封销售信。
- `XIABI_VERIFY_PAYMENT_CREATE=1`：真实创建一笔未支付微信 H5 年卡订单并检查 `h5Url`。
- `XIABI_VERIFY_PAID_ORDER_ID=订单ID`：真实检查已支付订单、权益流水、可选回调事件，并连续两次补权益确认幂等。
- `XIABI_VERIFY_SMS_PHONE=手机号`：真实发送一条短信验证码；收到后设置 `XIABI_VERIFY_SMS_CODE` 可复验绑定。如果同一轮也设置 `XIABI_VERIFY_DEEPSEEK=1`，会确认已生成/领取信件和首次免费权益归属到绑定用户。
- `XIABI_VERIFY_TTS=1`：真实调用 MiniMax TTS。
- `XIABI_VERIFY_ASR_AUDIO=本地音频路径`：真实调用服务端 ASR 接入槽。
- `XIABI_VERIFY_ASR_EXPECTED_TEXT=音频里应出现的文字`：可选；设置后会把转写结果和预期文字做归一化包含匹配，避免只验证“返回了任意文本”。
- `XIABI_VERIFY_WECHAT_VOICE=1`：真实调用微信 access_token/jsapi_ticket 接口，验证微信内 H5 语音所需 JS-SDK 签名配置。
- `XIABI_VERIFY_WECHAT_VOICE_MANUAL=1`：手机微信中按住说话已经人工确认可返回真实文本后再设置；不能替代前一项签名自检。

示例：

```bash
$env:XIABI_VERIFY_ADMIN_USERNAME="admin"
$env:XIABI_VERIFY_ADMIN_PASSWORD="你的后台密码"
$env:XIABI_PRODUCTION_STRICT="1"
npm run verify:production
```

## 当前仍需外部确认

- 真实微信小额支付必须由商户号完成一次真实付款和回调验收。
- 当前商户号需要确认 H5 支付产品权限；如果只计划微信内打开，则必须补齐 `WECHAT_MP_APP_SECRET`、公众号网页授权域名和支付授权目录，并按 JSAPI 支付验收。
- 真实短信必须用真实手机号验收一次，避免只验证接口结构。
- MiniMax 官方公开文档没有列出独立 ASR 接口；如必须服务端转写，请配置可用的 `VOICE_ASR_ENDPOINT`，或补充 MiniMax 对应接口文档后再精确适配。
- 服务端 ASR endpoint 配好后，必须先用 `XIABI_VERIFY_ASR_AUDIO` 和可选 `XIABI_VERIFY_ASR_EXPECTED_TEXT` 跑通真实音频验收，再设置 `VOICE_ASR_VERIFIED=1` 并重新部署/复验用户端语音入口。
- 如果走微信内 H5 语音输入，必须补齐 `WECHAT_MP_APP_SECRET`，在微信公众平台配置 JS 接口安全域名，并用 `XIABI_VERIFY_WECHAT_VOICE=1` 加手机微信实测完成验收；仅公开配置显示已配置不等于手机按住说话已经可用。
- 服务端已直出 PDF；当前导出包含可打印 HTML、TXT 文本版、DOCX 文档版和 PDF 文件，最终交付前需用 `XIABI_VERIFY_DEEPSEEK=1` 跑当前线上配置复验。
