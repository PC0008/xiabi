## 需要人工完成的 8 项

| 项目 | 为什么我无法继续自动验收 | 你需要做什么 |
| --- | --- | --- |
| 管理后台登录与运营接口 | 需要真实后台账号和密码，不能写进仓库或对话公开输出 | 在本机设置 `XIABI_VERIFY_ADMIN_USERNAME`、`XIABI_VERIFY_ADMIN_PASSWORD` 后运行 `npm run verify:production:report` |
| 微信支付下单 | 微信返回商户产品权限未开通，属于微信商户平台外部权限 | 在微信支付商户平台开通 H5 支付；微信内支付还需 JSAPI 支付和公众号网页授权配置 |
| 微信支付拉起审计链路 | 需要后台账号密码和一次真实创建微信订单 | 设置后台验收变量与 `XIABI_VERIFY_PAYMENT_CREATE=1` 后复验 |
| 微信真实付款回调与权益到账 | 需要真实付款订单和微信回调 | 完成一笔小额真实付款后，设置 `XIABI_VERIFY_PAID_ORDER_ID` 和 `XIABI_VERIFY_REQUIRE_WEBHOOK=1` 复验 |
| 短信发送与手机号绑定 | 需要可接收验证码的真实手机号，验证码只会发到手机 | 设置 `XIABI_VERIFY_SMS_PHONE` 发送验证码，收到后设置 `XIABI_VERIFY_SMS_CODE` 复验 |
| 短信发送审计链路 | 需要真实短信发送和后台账号 | 同一轮设置后台账号、手机号验收变量后复验 |
| 手机号绑定后资产归属 | 需要真实短信验证码，最好同一轮生成一封信 | 同一轮设置 `XIABI_VERIFY_DEEPSEEK=1`、手机号和验证码，确认信件/权益归属迁移 |
| 语音输入可用路径 | 需要可用 ASR endpoint 或手机微信 JS-SDK 实测 | 二选一：配置 `VOICE_ASR_ENDPOINT` 并用音频样本复验；或补 `WECHAT_MP_APP_SECRET`、公众号 JS 接口安全域名，并在手机微信按住说话确认返回文本 |

## 推荐人工验收顺序

1. 后台账号验收：先确认管理后台、配置传播、供应商自检可用。
2. DeepSeek + MiniMax TTS 复验：确认当前配置仍可真实生成和播放。
3. 短信验收：用真实手机号完成发码、绑定和资产归属。
4. 微信支付权限：先开通 H5/JSAPI 产品权限，再创建订单。
5. 真实付款验收：完成小额付款，确认回调和权益到账。
6. 语音输入验收：服务端 ASR 或微信内 JS-SDK 至少跑通一条。

## 常用命令

```powershell
npm run acceptance:inputs
npm run verify:preflight
npm run verify:production:report
npm run delivery:status
```

最终交付命令：

```powershell
npm run delivery:final
```

`delivery:final` 现在有硬门禁：即使部分验收命令返回 0，只要 `docs/production-readiness-latest.md` 没有显示 `完整可用：是`，它也会以非零码退出，防止误交付。
