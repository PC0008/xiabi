# AGENTS.md

本文件是《下笔有元》项目开发协作规则。所有代码修改、规划和交付都应遵守。

## 1. 当前项目口径

- 项目名：下笔有元。
- 用户端统一称呼：智多星。
- 用户端禁止出现：AI 分身、机器人、大模型、prompt、智能体、系统提示词、模型路由、向量记忆。
- 当前技术底座：手机端响应式 H5/Web + H5 总后台。
- 当前部署目标：Edgespark.dev。
- 当前不再按微信小程序测试号/CloudBase 推进；此前小程序、CloudBase 相关文档和代码只作为历史探索参考，除非用户明确要求恢复。
- 一期交互方案：通话感界面 + 按住说话互动，同时保留“切换打字模式”。
- 一期商业链路：先用 H5 主站验证，不展开多代理商 OEM。
- 一期后台：先做 H5 总后台，由总后台控制用户端文案、通话引导、销售信模板、价格权益和支付入口。
- 支付策略：H5 阶段先用 mock 流程；正式阶段按 Web/Edgespark.dev 可用的真实支付链路接入。不要默认使用小程序专用支付 API。
- 手机号策略：不在登录页强制获取；在“销售信排队生成/领取完整内容”节点引导授权或绑定。

## 2. 开发原则

- 先确定产品流程、UI 和数据契约，再进入大规模代码开发。
- 每个阶段只做当前闭环需要的功能，不提前实现代理商后台、一键 OEM、复杂 BI。
- 任何涉及订单、支付、权益、手机号、用户隐私的改动都要先说明方案。
- 所有权益只认订单流水和权益流水，不允许前端直接决定用户权限。
- 所有模型 Key、语音 Key、支付密钥只能保存在服务端。
- 即使一期只有主站，数据结构也要保留 `tenant_id`。
- 当前 H5 mock 可使用 `localStorage` 打通产品体验；正式数据、任务、支付、文件存储要按 Edgespark.dev 实际能力重新规划。
- 销售信生成、语音处理、PDF 导出等可能耗时的流程必须做成任务化、异步化、轮询或可恢复状态，不把前端同步等待当作稳定方案。
- 新增任何真实数据依赖前，必须同步给出需要创建的表/集合、索引、权限、初始配置和迁移说明，避免代码写完后运行时报数据结构不存在。

## 3. 工程结构规划

当前有效开发目标是 H5/Web：

- `h5/`：用户端 H5、H5 总后台、本地预览服务。
- `docs/`：产品、架构、UI、接口、数据模型文档。旧版小程序/CloudBase 文档需视为历史稿。
- `PROGRESS.md`：当前进度和下一步。
- `开发实施规划_v0.1.md`：早期开发总规划，涉及小程序/CloudBase 的内容需按当前口径重新审阅后再使用。
- `pages/`、`cloudfunctions/`：历史暂停的小程序/CloudBase 代码，不作为当前默认开发路径。

## 4. UI 开发规则

- UI 先出可打开原型，确认后再落真实 H5 页面。
- 用户端第一屏必须直接进入可用体验，不做营销站式落地页。
- 首页文案必须由总后台配置下发，前端只能保留默认兜底值，不能把主标题、副标题、按钮文案、首次免费提示、未领取提醒等写死。
- 用户端主视觉方向已定为“微信亲和绿”：首页采用极简直接布局，白底、粗标题、轻插画、绿色主按钮；通话页尽量贴近微信私人语音通话。
- 通话页底部只保留三个圆形按钮：左“扬声器”、中“按住说话”、右“挂断”；按钮下方放一行小字入口“切换打字模式”。
- 图标、通话按钮、精致小控件若用文字字符或纯 CSS 还原效果不佳，优先使用 SVG 图标实现，避免出现廉价、临时拼凑的视觉效果。
- 用户端要有“通话感”，但一期底层先用按住说话。
- 页面文案要围绕“智多星正在听、整理、写信、保存、领取”，不要暴露技术概念。
- 手机号授权文案要强调保存销售信、确认首次免费权益、生成完成提醒，不要写成粗暴强制。
- UI 审核基准图：`docs/assets/wechat-green-home-call-review-v1.png`，审核页：`docs/wechat-green-home-call-review.html`。

## 5. 关键验收标准

- H5 阶段：手机端响应式页面主流程可跑通，桌面浏览器预览不出现手机壳式容器。
- Mock 阶段：手机号、支付、权益、导出可模拟完整流转。
- 后台控制：H5 总后台配置可以影响用户端首页文案、通话引导、模板、价格权益和导出成交入口。
- Edgespark.dev 阶段：用户端 H5 和 H5 总后台可以通过公网地址打开，并完成基础路由、静态资源、配置读取验证。
- 正式支付阶段：真实支付、真实回调、真实权益发放可验证。
- 支付回调必须幂等，重复回调不能重复发权益。
- 生成失败不能丢失用户已采集信息。
- 任何长耗时后端任务，都必须能保存进度并允许前端查询状态。
- 每次新增真实数据库/存储/队列依赖前，先给出 Edgespark.dev 对应的创建清单、索引/权限和初始化配置。
